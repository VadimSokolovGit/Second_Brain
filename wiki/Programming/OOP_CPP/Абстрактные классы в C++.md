# Абстрактные классы в C++

> [!info] Определение
> **Абстрактный класс** — это класс, описывающий общую концепцию или интерфейс, но сам по себе являющийся неполным. Из-за этого C++ **запрещает создавать объекты** такого класса напрямую (например, нельзя написать `Instrument i;`).

> [!warning] Правило абстрактности
> Класс автоматически становится абстрактным, если в нём объявлена хотя бы одна **чистая виртуальная функция**.

## Чистая виртуальная функция

> [!info] Определение
> **Чистая виртуальная функция** (*pure virtual function*) — это виртуальная функция, которая по умолчанию не имеет реализации в базовом классе. Синтаксически она заканчивается на `= 0`:

```cpp
virtual void play() const = 0;
```
> [!note] Смысл
> Это означает: "У этого метода нет кода, это лишь декларация намерения. Любой класс-наследник обязан переопределить эту функцию и написать для неё собственную реализацию".

## Зачем это нужно? (Контракт)

Абстрактные классы используются для задания **интерфейсов** (строгих контрактов на поведение).

- **Гарантия реализации:** Наследники не могут "забыть" реализовать требуемый метод. Иначе программа просто не скомпилируется (наследник тоже будет считаться абстрактным классом).
- **Динамический полиморфизм:** Основная система может управлять любыми производными классами (например, `Guitar`, `Piano`) через единый указатель или ссылку на абстрактную базу (например, `Instrument*`), не зная о конкретных типах объектов под капотом.
- **Масштабируемость:** Вы можете добавлять новые сущности (наследников) без изменения основного кода, который работает с базовым классом.

> [!example] Пример реализации
> 
> ```cpp
> #include <iostream>
> #include <vector>
> 
> // АБСТРАКТНЫЙ КЛАСС (Интерфейс)
> class Instrument {
> public:
>     // Чистая виртуальная функция. Делает класс абстрактным.
>     virtual void play() const = 0; 
> 
>     // Виртуальный деструктор. Обязателен для абстрактных классов!
>     virtual ~Instrument() {
>         std::cout << "Деструктор абстрактного Инструмента\n";
>     }
> };
> 
> // Конкретный класс-наследник 1
> class Guitar : public Instrument {
> public:
>     // Обязаны переопределить чистую виртуальную функцию
>     void play() const override {
>         std::cout << "Бринь-бринь! Играет гитара.\n";
>     }
>     
>     ~Guitar() override {
>         std::cout << "Деструктор Гитары\n";
>     }
> };
> 
> // Конкретный класс-наследник 2
> class Piano : public Instrument {
> public:
>     // Обязаны переопределить чистую виртуальную функцию
>     void play() const override {
>         std::cout << "Ля-ля-ля! Играет пианино.\n";
>     }
>     
>     ~Piano() override {
>         std::cout << "Деструктор Пианино\n";
>     }
> };
> 
> int main() {
>     // line ниже вызовет ошибку: " cannot declare variable 'i' to be of abstract type 'Instrument' "
>     // Instrument i; 
> 
>     // Но мы МОЖЕМ создавать указатели и ссылки на абстрактный класс!
>     // Это основа динамического полиморфизма.
>     std::vector<Instrument*> orchestra;
>     
>     orchestra.push_back(new Guitar());
>     orchestra.push_back(new Piano());
> 
>     std::cout << "--- Оркестр, играй! ---\n";
>     for (const Instrument* inst : orchestra) {
>         inst->play(); // Полиморфный вызов
>     }
> 
>     std::cout << "\n--- Распускаем оркестр ---\n";
>     for (Instrument* inst : orchestra) {
>         delete inst; // Корректно удалит и гитару/пианино, и базу благодаря virtual ~Instrument
>     }
> 
>     return 0;
> }
> ```
