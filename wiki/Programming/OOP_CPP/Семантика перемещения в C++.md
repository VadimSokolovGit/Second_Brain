# Семантика перемещения в C++

> [!info] Суть концепции
> **Семантика перемещения** (*Move semantics*) была введена в C++11 для устранения лишних затрат на копирование больших объёмов памяти. 
> Вместо полного побайтового копирования данных, ресурсы (указатели на динамическую память) просто **"перекидываются"** от старого объекта к новому. Старый объект при этом становится пустым.

## Rvalue-ссылки (`&&`)

Чтобы компилятор знал, когда можно безопасно «обокрасть» объект, используются **rvalue-ссылки**.

- **lvalue-ссылка** (`Type&`) привязывается к постоянным, именованным объектам.
- **rvalue-ссылка** (`Type&&`) привязывается к временным, безымянным объектам, которые вот-вот будут уничтожены (например, результат функции). Поскольку объект всё равно исчезнет, его ресурсы можно забрать.

> [!note] Принудительное перемещение
> Функция `std::move()` позволяет явно сказать компилятору: "я больше не буду использовать этот объект, его можно перемещать". Она принудительно превращает `lvalue` в `rvalue`.

## Конструктор перемещения (Move Constructor)

Создаёт новый объект, забирая ресурсы у временного.

> [!example] Алгоритм реализации конструктора:
> 1. **Копируем** указатели/данные из `other` в текущий объект (`this`).
> 2. **Обязательно обнуляем** указатели в `other`. Если этого не сделать, деструктор `other` при своём уничтожении удалит украденную нами память.
> 3. Обязателен спецификатор **`noexcept`** — гарантирует отсутствие исключений (это критично для оптимизаций стандартных контейнеров, таких как `std::vector`).

## Оператор присваивания перемещением (Move Assignment)

Используется, когда оба объекта уже существуют, но данные переносятся из временного (правого) в левый.

> [!example] Алгоритм реализации оператора:
> 1. **Проверка на самоприсваивание** (`if (this == &other)`).
> 2. **Очистка собственных старых ресурсов** (чтобы избежать утечки).
> 3. **Копирование** указателей из `other` в `this`.
> 4. **Обнуление** указателей в `other`.
> 5. **Возврат** `*this`.

> [!example] Пример: Класс с перемещением
> ```cpp
> #include <iostream>
> #include <utility> // Для std::move
> 
> class Buffer {
> private:
>     int* data;
>     size_t size;
> 
> public:
>     // Обычный конструктор
>     Buffer(size_t s) : size(s) {
>         data = new int[size];
>         std::cout << "Создан буфер\n";
>     }
> 
>     // Деструктор
>     ~Buffer() {
>         delete[] data;
>         std::cout << "Буфер уничтожен\n";
>     }
> 
>     // 1. КОНСТРУКТОР ПЕРЕМЕЩЕНИЯ
>     Buffer(Buffer&& other) noexcept {
>         data = other.data;   // Забираем указатель на память
>         size = other.size;   // Забираем размер
> 
>         other.data = nullptr; // ОБЯЗАТЕЛЬНО: обнуляем старый объект
>         other.size = 0;
>         std::cout << "Сработал конструктор ПЕРЕМЕЩЕНИЯ\n";
>     }
> 
>     // 2. ОПЕРАТОР ПРИСВАИВАНИЯ ПЕРЕМЕЩЕНИЕМ
>     Buffer& operator=(Buffer&& other) noexcept {
>         if (this == &other) return *this; // Проверка на самоприсваивание
> 
>         delete[] data; // Очищаем свою старую память
> 
>         data = other.data; // Забираем ресурсы
>         size = other.size;
> 
>         other.data = nullptr; // Обнуляем старый объект
>         other.size = 0;
>         std::cout << "Сработал оператор присваивания ПЕРЕМЕЩЕНИЕМ\n";
> 
>         return *this;
>     }
> };
> 
> int main() {
>     Buffer buf1(100); // Обычный конструктор
>     
>     // std::move() принудительно превращает buf1 из lvalue в rvalue,
>     // заставляя компилятор вызвать перемещение вместо копирования
>     Buffer buf2 = std::move(buf1); // Вызовется конструктор перемещения
> 
>     // Теперь buf1 пустой (его data == nullptr), а buf2 владеет массивом.
>     return 0;
> }
> ```
