# Контейнер std::map в C++

> [!info] Теоретическая база
> В отличие от векторов и списков, где элементы ищутся по их порядковому индексу, в **ассоциативных контейнерах** поиск происходит по ключу. Самым ярким представителем является `std::map`.
> 
> **`std::map`** — это словарь (ассоциативный массив), хранящий элементы в виде пар "Ключ — Значение" (`std::pair<const Key, T>`).
> - **Ключ (Key):** Уникальный идентификатор (логин, номер телефона и т.д.). Дубликаты запрещены.
> - **Значение (Value):** Информация, привязанная к ключу.

## Структура в памяти (Красно-чёрное дерево)

> [!abstract] Устройство под капотом
> Внутри `std::map` построен на основе **Красно-черного дерева (Red-Black Tree)**, которое является сбалансированным бинарным деревом поиска.
> 
> **Следствия такой архитектуры:**
> 1. Все элементы *всегда автоматически отсортированы* по возрастанию ключа.
> 2. Поиск, добавление и удаление элементов работают за логарифмическое время **$O(\log n)$**. Даже в массиве из миллиона записей нужный ключ найдётся всего за ~20 шагов сравнения.

## Особенности оператора `[]`

У `std::map` есть уникальная особенность работы с квадратными скобками:

> [!success] Удобное добавление
> При вызове `my_map[новый_ключ] = новое_значение;` контейнер понимает, что ключа ещё нет, и *мгновенно создаёт новую пару*, помещая её в дерево.

> [!warning] Ловушка для экзамена
> Если вы попытаетесь вывести значение по несуществующему ключу (например, `std::cout << my_map["несуществующий_ключ"];`), `std::map` **автоматически создаст этот ключ** в словаре и запишет туда пустое/дефолтное значение (например, `0` для чисел). 
> 
> *Как избежать:* Для проверки наличия ключа без его случайного создания используйте методы `.count(key)` (возвращает 1 или 0) или `.find(key)`.

## Сравнение контейнеров для экзамена

> [!important] Краткий итог
> - **`std::vector`** — быстрый доступ по индексу, элементы лежат подряд, данные не сортированы.
> - **`std::list`** — быстрая вставка в любое место, элементы связаны указателями, данные не сортированы.
> - **`std::map`** — быстрый доступ по любому типу ключа, элементы в памяти лежат в виде дерева, данные всегда отсортированы.

## Пример кода: Телефонная книга

> [!example] Код
> ```cpp
> #include <iostream>
> #include <string>
> #include <map> // Обязательный заголовочный файл
> 
> int main() {
>     setlocale(LC_ALL, "Russian");
> 
>     // Создаем словарь, где Ключ - string, Значение - long long
>     std::map<std::string, long long> phoneBook;
> 
>     // 1. Добавление элементов (Способ 1 — через оператор [])
>     phoneBook["Иван"] = 89991112233;
>     phoneBook["Анна"] = 89995556677;
>     phoneBook["Борис"] = 89993334455;
> 
>     // 2. Добавление элементов (Способ 2 — классический insert через пару)
>     phoneBook.insert(std::make_pair("Елена", 89998889900));
> 
>     // 3. Вывод на экран
>     // Обрати внимание: при обходе элементы будут отсортированы по алфавиту (по Ключу)!
>     // Анна -> Борис -> Елена -> Иван
>     std::cout << "--- Список контактов (автоматически отсортирован) ---\n";
>     for (const auto& pair : phoneBook) {
>         // pair.first — это Ключ, pair.second — это Значение
>         std::cout << pair.first << ": " << pair.second << "\n";
>     }
>     std::cout << "\n";
> 
>     // 4. Поиск элемента
>     std::string nameForSearch = "Борис";
>     
>     // Метод count(key) возвращает 1, если ключ есть, и 0, если его нет
>     if (phoneBook.count(nameForSearch)) {
>         std::cout << "Номер " << nameForSearch << ": " << phoneBook[nameForSearch] << "\n";
>     } else {
>         std::cout << "Контакт не найден.\n";
>     }
> 
>     // 5. Удаление элемента по ключу
>     phoneBook.erase("Иван");
> 
>     return 0;
> }
> ```
