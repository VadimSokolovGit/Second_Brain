# Алгоритмы STL в C++

> [!info] 🎯 Определение
> Библиотека `<algorithm>` предоставляет набор готовых и оптимизированных шаблонов функций для работы с контейнерами. В этой заметке собраны ключевые алгоритмы: кастомная сортировка, проверка подмножеств и ряд продвинутых утилит (поиск, разность, сдвиг, трансформация).

## 1. Кастомная сортировка (`std::sort`)

Стандартный алгоритм `std::sort` по умолчанию сортирует элементы по возрастанию. Однако мы можем передать собственный **предикат** для изменения логики (например, для сортировки сложных объектов). Предикат должен возвращать `true`, если первый элемент (`a`) должен стоять в результирующем массиве **строго раньше** второго (`b`).

> [!warning] Строгий слабый порядок (Strict Weak Ordering)
> Предикат сортировки **обязан** подчиняться правилу строгого слабого порядка. Если его нарушить, программа может упасть (SegFault). Для двух абсолютно одинаковых элементов предикат обязан возвращать `false` (то есть использовать строгое сравнение `return a < b;`, а не `a <= b;`).

```cpp
// Пример предиката для сортировки студентов по убыванию рейтинга
bool compareStudents(const Student& a, const Student& b) {
    if (a.rating != b.rating) return a.rating > b.rating; 
    return a.name < b.name;
}
// Использование: std::sort(group.begin(), group.end(), compareStudents);
```

## 2. Проверка подмножеств (`std::includes`)

`std::includes` проверяет, является ли второй диапазон подмножеством первого («Содержит ли контейнер A внутри себя абсолютно все элементы из контейнера B?»).

- **Обязательная сортировка**: Требует, чтобы оба контейнера были **строго отсортированы** перед вызовом.
- **Учет дубликатов**: Корректно обрабатывает повторяющиеся элементы. Если в B две двойки, в A должно быть минимум две двойки.
- **Возвращаемое значение**: `bool` (`true`/`false`).

```cpp
std::sort(A.begin(), A.end());
std::sort(B.begin(), B.end());
bool containsAll = std::includes(A.begin(), A.end(), B.begin(), B.end());
```

## 3. Поиск границ (`std::equal_range`)
**Что делает:** Быстро находит поддиапазон одинаковых элементов, эквивалентных заданному значению.
**Синтаксис:** `auto range = std::equal_range(v.begin(), v.end(), target);`
- Объединяет в себе `std::lower_bound` и `std::upper_bound`.
- Возвращает `std::pair` из двух итераторов на начало и конец найденного поддиапазона.
- **Требует предварительной сортировки** контейнера ($O(\log n)$).

## 4. Разность множеств (`std::set_difference`)
**Что делает:** Находит математическую разность двух множеств (элементы, которые есть в $A$, но отсутствуют в $B$).
**Синтаксис:** `std::set_difference(A.begin(), A.end(), B.begin(), B.end(), std::back_inserter(result));`
- Вычисляет элементы строго из диапазона $A \setminus B$. Учитывает дубликаты.
- **Требует сортировки обоих контейнеров** перед вызовом ($O(n + m)$).

## 5. Циклический сдвиг (`std::rotate`)
**Что делает:** Выполняет циклический сдвиг элементов внутри диапазона, переставляя их местами вокруг выбранной точки.
**Синтаксис:** `std::rotate(v.begin(), middle_iterator, v.end());`
- Буквально «снимает верхушку» массива до элемента `middle` и перекладывает её в самый конец. 
- Операция выполняется за линейное время и работает на месте (in-place).

## 6. Поиск расхождений (`std::mismatch`)
**Что делает:** Параллельно сравнивает две последовательности шаг за шагом и находит самое первое место, где их элементы перестают совпадать.
**Синтаксис:** `auto res = std::mismatch(L1.begin(), L1.end(), L2.begin());`
- Возвращает `std::pair` итераторов, указывающих на точки расхождения. Если контейнеры идентичны, вернется `.end()`.

## 7. Трансформация (`std::transform`)
**Что делает:** Последовательно обходит элементы контейнера, применяет к каждому из них заданную функцию (лямбду/функтор) и сохраняет измененный результат.
**Синтаксис:** `std::transform(L.begin(), L.end(), L.begin(), [](int x){ return x * 2; });`
- Выполняет роль классической функции отображения (`map`) из функционального программирования. Может работать in-place.

## 8. Заполнение диапазона (`std::fill`)
**Что делает:** Заполняет весь заданный диапазон копиями указанного значения.
**Синтаксис:** `std::fill(iterator_first, iterator_last, value);`
- Последовательно перебирает элементы от начального итератора до конечного и присваивает каждому `value`.
- **Сложность:** Строго линейная $O(n)$.

## 9. Заполнение N элементов (`std::fill_n`)
**Что делает:** Похож на `std::fill`, но принимает количество элементов (`count`), которое нужно заполнить, начиная с позиции `first`.
**Синтаксис:** `std::fill_n(iterator_first, count, value);`

> [!warning] Важный нюанс безопасности
> Программист обязан гарантировать, что в контейнере физически существует как минимум `count` элементов. Если попытаться заполнить больше, произойдет выход за границы памяти (Undefined Behavior).
> **Исключение:** Можно динамически добавить новые элементы в пустой контейнер, если передать в качестве первого аргумента итератор вставки `std::back_inserter`.

```cpp
// 1. std::fill (весь вектор)
std::vector<int> v1 = { 1, 2, 3, 4, 5, 6 };
std::fill(v1.begin(), v1.end(), 9); // 9 9 9 9 9 9

// 2. std::fill_n (первые 3 элемента)
std::vector<int> v2 = { 10, 20, 30, 40, 50 };
std::fill_n(v2.begin(), 3, -1); // -1 -1 -1 40 50

// 3. Безопасное расширение через back_inserter
std::vector<int> v3; // Абсолютно пустой вектор
std::fill_n(std::back_inserter(v3), 4, 77); // 77 77 77 77
```

## 10. Генерация последовательностей (`std::generate`)
**Что делает:** Последовательно обходит элементы заданного диапазона и присваивает каждому результат вызова генератора (функции, лямбды или функтора, возвращающего значение без аргументов).
**Синтаксис:** `std::generate(v.begin(), v.end(), generator_function);`
- **Сложность:** Строго линейная $O(n)$.
- **Суперсила:** В отличие от `std::fill`, генератор может хранить внутри себя состояние (например, лямбда с захватом переменной по ссылке). Это позволяет динамически создавать сложные последовательности: прогрессии, случайные числа, ряды Фибоначчи и т.д.

```cpp
// Пример: генерация последовательности четных чисел (2, 4, 6, 8, 10...)
std::vector<int> numbers(5); // Вектор заранее должен иметь размер
int currentEven = 0;

// Лямбда захватывает currentEven по ссылке (&) и обновляет его при каждом вызове
std::generate(numbers.begin(), numbers.end(), [&currentEven]() {
    currentEven += 2;
    return currentEven;
});
```
