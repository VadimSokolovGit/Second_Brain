# Хранение полиморфных объектов в контейнерах (C++)

При работе с полиморфными иерархиями возникает архитектурный вопрос: как правильно хранить группу разнородных объектов-наследников (например, в массиве или векторе).

> [!warning] Проблема «Срезки» объектов (Object Slicing)
> Если попытаться создать контейнер напрямую из базового класса (`std::vector<Base>`) и положить туда объекты наследников (например, `Derived`), произойдёт **срезка** (slicing).
> 
> Вектор выделит память строго под размер `Base`. Все уникальные поля и данные `Derived` будут отрезаны и безвозвратно утеряны. Полиморфизм работать не будет.

## Правильное решение
Полиморфные объекты обязаны храниться в контейнерах **только по указателям**. Указатель на 64-битной системе всегда занимает 8 байт, поэтому контейнер хранит одинаковые адреса, ведущие на полноценные объекты в куче.

> [!success] Варианты хранения
> 1. **Сырые указатели** (`std::vector<Base*>`): Требуют ручного обхода контейнера перед его удалением (через цикл) с вызовом `delete` для каждого элемента во избежание утечек.
> 2. **Умные указатели** (`std::vector<std::unique_ptr<Base>>`): Современный подход (C++11+). Автоматически очищают память объектов при удалении контейнера.

## Пример с сырыми указателями

> [!example] Код
> ```cpp
> #include <iostream>
> #include <vector>
> #include <string>
> 
> // Базовый класс
> class Employee {
> protected:
>     std::string name;
> public:
>     Employee(std::string n) : name(n) {}
>     
>     // Виртуальный деструктор обязателен для корректного удаления из контейнера!
>     virtual ~Employee() {
>         std::cout << "~Employee (база для " << name << " уничтожена)\n";
>     }
> 
>     virtual void work() const = 0; // Чистая виртуальная функция
> };
> 
> // Наследник 1
> class Developer : public Employee {
> public:
>     Developer(std::string n) : Employee(n) {}
>     ~Developer() override { std::cout << "~Developer очищен\n"; }
>     
>     void work() const override {
>         std::cout << "Разработчик " << name << " пишет код на C++.\n";
>     }
> };
> 
> // Наследник 2
> class Manager : public Employee {
> public:
>     Manager(std::string n) : Employee(n) {}
>     ~Manager() override { std::cout << "~Manager очищен\n"; }
>     
>     void work() const override {
>         std::cout << "Менеджер " << name << " раздает задачи команде.\n";
>     }
> };
> 
> int main() {
>     std::vector<Employee*> staff;
> 
>     // Безопасно помещаем разные объекты в один вектор указателей
>     staff.push_back(new Developer("Алексей"));
>     staff.push_back(new Manager("Елена"));
> 
>     // Полиморфный вызов (работает!)
>     for (const Employee* emp : staff) {
>         emp->work(); 
>     }
> 
>     // Ручная очистка динамической памяти (обязательно для сырых указателей)
>     for (size_t i = 0; i < staff.size(); ++i) {
>         delete staff[i]; 
>     }
>     staff.clear();
> 
>     return 0;
> }
> ```
