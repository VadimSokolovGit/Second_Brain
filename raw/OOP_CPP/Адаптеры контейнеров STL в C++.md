# Адаптеры контейнеров STL в C++

В стандартной библиотеке шаблонов (STL) существует особый класс сущностей — **адаптеры контейнеров**.

> [!info] Что такое адаптер контейнера?
> Адаптер сам по себе не хранит данные и не реализует низкоуровневую структуру. Он берет уже существующий базовый контейнер (например, `std::vector` или `std::deque`) и «ограничивает» его интерфейс, предоставляя строгий специфичный набор методов. 
> *Смысл:* скрыть лишние возможности базового контейнера и заставить структуру работать по жестким правилам.

## Основные адаптеры в STL

| Адаптер | Принцип работы | Описание | Базовый контейнер по умолчанию |
|---|---|---|---|
| `std::stack` | LIFO | Стек («последним пришел — первым ушел»). Доступ только к вершине. | `std::deque` |
| `std::queue` | FIFO | Очередь («первым пришел — первым ушел»). Доступ только к началу и концу. | `std::deque` |
| `std::priority_queue` | Очередь с приоритетом | Элементы автоматически сортируются так, что на выходе всегда самый «большой» (или приоритетный). | `std::vector` |

> [!tip] Изменение базового контейнера (Underlying Container)
> Можно явно указать, какой контейнер должен работать "под капотом":
> ```cpp
> // Стек, который внутри использует std::vector вместо std::deque
> std::stack<int, std::vector<int>> myStack; 
> ```

## Важный теоретический нюанс (Для экзамена)

> [!warning] Почему у адаптеров нет итераторов?
> У классов `std::stack`, `std::queue` и `std::priority_queue` принципиально **запрещены итераторы** (отсутствуют методы `.begin()` и `.end()`). 
> *Причина:* Если бы они были, программист мог бы получить доступ в середину стека или очереди, что нарушило бы фундаментальные математические правила этих структур (LIFO/FIFO).

## Пример работы (Стек и Очередь)

> [!example] Код
> ```cpp
> #include <iostream>
> #include <stack>
> #include <queue>
> 
> int main() {
>     setlocale(LC_ALL, "Russian");
> 
>     // --- 1. СТЕК (LIFO) ---
>     std::stack<int> s;
>     s.push(10);
>     s.push(20);
>     s.push(30); // 30 на вершине стека
> 
>     std::cout << "Вершина стека: " << s.top() << "\n"; // 30
> 
>     std::cout << "Извлекаем: ";
>     while (!s.empty()) {
>         std::cout << s.top() << " ";
>         s.pop();
>     } // Вывод: 30 20 10
> 
> 
>     // --- 2. ОЧЕРЕДЬ (FIFO) ---
>     std::queue<std::string> q;
>     q.push("Первый");
>     q.push("Второй");
>     q.push("Третий"); // Встает в конец
> 
>     std::cout << "\nПервый: " << q.front() << ", Последний: " << q.back() << "\n";
> 
>     std::cout << "Извлекаем: ";
>     while (!q.empty()) {
>         std::cout << q.front() << " ";
>         q.pop();
>     } // Вывод: Первый Второй Третий
> 
>     return 0;
> }
> ```
