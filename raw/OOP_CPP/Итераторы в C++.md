# Итераторы в C++

> [!info] 🎯 Определение
> **Итератор** — это специальный объект библиотеки STL, который концептуально представляет собой абстрактный указатель на элемент контейнера. Он служит универсальным мостом между структурами данных (контейнерами) и алгоритмами.

## ❓ Зачем нужны итераторы?

Разные контейнеры устроены в памяти совершенно по-разному (`std::vector` — сплошной массив, `std::list` — разбросанные по куче узлы). Без итераторов для каждого контейнера пришлось бы писать свои версии алгоритмов (поиска, сортировки и т.д.).

Итератор скрывает внутреннее устройство контейнера и предоставляет **единый стандартный интерфейс** для перебора. Для любого итератора всегда работают три базовые операции:
1. `*it` (разыменование) — доступ к элементу, на который он указывает.
2. `++it` (инкремент) — сдвиг итератора на следующий элемент.
3. `==` и `!=` (сравнение) — проверка, не совпадает ли наш итератор с другим.

## 📍 Главные маркеры: `.begin()` и `.end()`

Любой стандартный контейнер умеет возвращать два базовых итератора:
- `.begin()` — указывает на самый первый элемент контейнера.
- `.end()` — указывает на мнимую позицию **ЗА** последним элементом (так называемый барьерный элемент). 

> [!warning] Важно!
> Разыменовывать итератор `.end()` категорически запрещено — это приведет к падению (undefined behavior). Он нужен исключительно как маркер остановки цикла.

## 💻 Пример использования

С итераторами мы можем обходить массив (`std::vector`) и двусвязный список (`std::list`), используя абсолютно одинаковый синтаксис:

```cpp
#include <iostream>
#include <vector>
#include <list>

int main() {
    // 1. Обход вектора через итераторы
    std::vector<int> v = { 10, 20, 30 };
    std::cout << "Элементы вектора: ";
    
    for (std::vector<int>::iterator it = v.begin(); it != v.end(); ++it) {
        std::cout << *it << " "; 
    }
    std::cout << "\n";

    // 2. Обход списка через итераторы (Синтаксис идентичен!)
    std::list<std::string> myList = { "Раз", "Два", "Три" };
    std::cout << "Элементы списка:  ";
    
    // В современном C++ вместо длинного типа пишут просто auto
    for (auto it = myList.begin(); it != myList.end(); ++it) {
        std::cout << *it << " "; 
    }
    std::cout << "\n";

    // 3. Изменение значения через итератор
    auto firstElementIt = v.begin();
    *firstElementIt = 99; // Меняем 10 на 99 (запись по итератору)
    
    return 0;
}
```

## 🗂 5 Категорий (Иерархия) Итераторов

Итераторы в C++ разделены на 5 основных категорий на основе концепции «силы» (количества поддерживаемых операций). Это сделано ради оптимизации алгоритмов STL: если контейнер позволяет прыгать по памяти во все стороны, алгоритм выберет быструю сортировку; если контейнер слабый — алгоритм подстроится. 

Иерархия выглядит следующим образом (от слабых к сильным):

### 1. Итераторы ввода (Input Iterators)
Самая слабая категория. Позволяют только последовательное чтение данных и движение строго вперед. Читать один и тот же элемент дважды или возвращаться назад нельзя.
- **Доступные операции:** `++it`, `*it` (только для чтения), `==`, `!=`.
- **Пример:** `std::istream_iterator` (поток ввода данных из консоли или файла).

### 2. Итераторы вывода (Output Iterators)
Зеркальный аналог итераторов ввода. Позволяют только последовательную запись данных и движение строго вперед. Прочитать записанное через этот же итератор нельзя.
- **Доступные операции:** `++it`, `*it = value` (только для записи).
- **Пример:** `std::ostream_iterator`, `std::back_insert_iterator` (создается через `std::back_inserter`).

### 3. Однонаправленные итераторы (Forward Iterators)
Объединяют возможности ввода и вывода. Позволяют читать и писать данные, многократно проходить по одному и тому же диапазону, но двигаться могут **только вперед**.
- **Доступные операции:** Все операции ввода/вывода + возможность многократного прохода по элементам.
- **Пример:** Итераторы контейнера `std::forward_list` (односвязный список), `std::unordered_set`.

### 4. Двунаправленные итераторы (Bidirectional Iterators)
Обладают всеми свойствами однонаправленных, но получают суперсилу — **умеют делать шаг назад** (`--it`).
- **Доступные операции:** Все операции Forward Iterator + декремент `--it`.
- **Пример:** Итераторы контейнеров `std::list` (двусвязный список), `std::set`, `std::map`. 
> [!warning] Важно
> На них нельзя применить арифметику (например, написать `it + 5` нельзя, нужно делать `++` пять раз).

### 5. Итераторы произвольного доступа (Random Access Iterators)
Самая мощная категория. Ведут себя в памяти как абсолютно полноценные указатели. Позволяют мгновенно **прыгать на любой элемент за константное время $O(1)$** с помощью арифметики.
- **Доступные операции:** Все операции Bidirectional + арифметика (`it + n`, `it - n`, `it += n`), индексация `it[n]`, операции сравнения порядков (`<`, `>`, `<=`, `>=`).
- **Пример:** Итераторы контейнеров `std::vector`, `std::deque`, обычные встроенные массивы типа `int arr[]`.

> [!tip] Шпаргалка для ответа на зачете
> «Категория итератора определяется архитектурой самого контейнера в памяти. К примеру, `std::vector` лежит в памяти непрерывным куском, поэтому его итератор — произвольного доступа. А `std::list` разбросан по куче отдельными узлами, из-за чего мы можем ходить по нему только по цепочке шагами вперед-назад, что делает его итератор двунаправленным».
