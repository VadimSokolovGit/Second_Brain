# Перегрузка операторов в C++

> [!info] Суть перегрузки
> **Перегрузка операторов** позволяет задать собственное поведение стандартным операторам (таким как `+`, `-`, `*`, `==`) для пользовательских классов. Однако язык накладывает строгие ограничения на то, что можно, а что нельзя делать.

## Критические правила перегрузки

> [!warning] Что делать ЗАПРЕЩЕНО
> 1. **Нельзя придумывать новые символы.** Разрешено перегружать только существующие в C++ операторы (например, нельзя создать оператор `**` для возведения в степень).
> 2. **Нельзя изменять приоритет или ассоциативность.** Умножение `*` всегда будет выполняться раньше сложения `+`, независимо от вашей реализации.
> 3. **Нельзя перегружать поведение встроенных типов.** Вы не можете переписать поведение `int + int`. Хотя бы один из операндов *обязан* быть пользовательским объектом (классом).

## Запрещенные к перегрузке операторы

> [!important] 4 оператора, которые перегрузить нельзя
> Эти операторы строго зашиты в компилятор, и их поведение изменить невозможно:
> - **`.`** — доступ к членам класса (member access).
> - **`.*`** — доступ к членам класса по указателю (pointer to member).
> - **`::`** — разрешение области видимости (scope resolution).
> - **`?:`** — тернарный условный оператор.

## Два пути перегрузки

Оператор можно перегрузить двумя способами, в зависимости от ситуации:

1. **Как метод класса** (*Member function*):
   Левый операнд неявно передается через указатель `this`, а правый идет как параметр функции. Подходит, когда левым операндом является объект вашего класса.
2. **Как внешнюю функцию** (часто дружественную — `friend`):
   Функция живет вне класса, и ей явно передаются оба операнда (и левый, и правый). Это необходимо, если левым операндом выступает объект чужого класса (например, поток вывода `std::ostream` при перегрузке оператора `<<`).

> [!example] Пример перегрузки (методы и внешние функции)
> ```cpp
> #include <iostream>
> 
> class Vector2D {
> private:
>     double x;
>     double y;
> 
> public:
>     Vector2D(double x_val = 0.0, double y_val = 0.0) : x(x_val), y(y_val) {}
> 
>     // --- 1. Перегрузка функции (метода) внутри класса ---
>     void print() const {
>         std::cout << "(" << x << ", " << y << ")\n";
>     }
>     // Перегруженная версия метода (с другим списком параметров)
>     void print(const std::string& prefix) const {
>         std::cout << prefix << ": (" << x << ", " << y << ")\n";
>     }
> 
>     // --- 2. Перегрузка оператора + как МЕТОДА КЛАССА ---
>     // Левый операнд — это this, правый — other
>     Vector2D operator+(const Vector2D& other) const {
>         return Vector2D(this->x + other.x, this->y + other.y);
>     }
> 
>     // Объявляем внешнюю функцию другом (friend), чтобы дать доступ к private полям x и y
>     friend std::ostream& operator<<(std::ostream& os, const Vector2D& vec);
> };
> 
> // --- 3. Перегрузка оператора << как ВНЕШНЕЙ ФУНКЦИИ ---
> // Позволяет писать std::cout << vec вместо vec.print()
> std::ostream& operator<<(std::ostream& os, const Vector2D& vec) {
>     os << "[" << vec.x << "; " << vec.y << "]";
>     return os; // Возвращаем поток по ссылке для цепочки вызовов
> }
> 
> int main() {
>     Vector2D v1(1.5, 2.0);
>     Vector2D v2(3.0, 4.5);
> 
>     // Демонстрация перегрузки функций
>     v1.print();                 // Выведет: (1.5, 2)
>     v1.print("Вектор 1");       // Выведет: Вектор 1: (1.5, 2)
> 
>     // Демонстрация перегрузки операторов
>     Vector2D v3 = v1 + v2;      // Эквивалентно: v1.operator+(v2)
>     
>     // Красивый вывод через перегруженный оператор <<
>     std::cout << "Результат сложения v3 = " << v3 << std::endl; 
>     // Выведет: Результат сложения v3 = [4.5; 6.5]
> 
>     return 0;
> }
> ```
