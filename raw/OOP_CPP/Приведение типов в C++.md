# Приведение типов в C++ (Casting)

> [!info] Суть проблемы
> В языке Си для приведения типов использовался синтаксис `(Тип)выражение`. В C++ такой подход считается опасным, так как он заставляет компилятор интерпретировать память по-новому, игнорируя безопасность.
> Вместо этого в C++ ввели **четыре специальных оператора** явного приведения типов.

## 4 вида приведения типов в C++

> [!abstract] 1. `static_cast` (Статическое приведение)
> Используется для стандартных, безопасных и логически понятных компилятору преобразований на этапе компиляции.
> - Конвертация базовых типов (например, `double` в `int`).
> - Приведение указателя на наследника к указателю на базовый класс (**Upcasting**).
> - Не выполняет проверок во время выполнения (в рантайме).

> [!abstract] 2. `dynamic_cast` (Динамическое приведение)
> Используется исключительно для безопасного перемещения по полиморфной иерархии наследования.
> - Позволяет безопасно спуститься от базы к наследнику (**Downcasting**).
> - Работает во время выполнения программы (в рантайме).
> - **Требование:** Базовый класс должен быть полиморфным (иметь хотя бы одну виртуальную функцию).
> - Если приведение невозможно (объекты не родственны), возвращает `nullptr`. Если приводится ссылка, генерирует исключение `std::bad_cast`.

> [!abstract] 3. `const_cast` (Приведение константности)
> Единственный оператор, который умеет добавлять или снимать модификаторы `const` и `volatile`.
> - **Внимание:** Изменение изначально константного объекта через `const_cast` ведет к неопределенному поведению. Он нужен только для передачи константных переменных в старые API, которые принимают неконстантные параметры, но фактически данные не меняют.

> [!warning] 4. `reinterpret_cast` (Низкоуровневое переинтерпретирование)
> Самый опасный каст. Он заставляет компилятор смотреть на биты в памяти одного типа так, будто это совершенно другой тип (например, превратить адрес указателя на объект в массив `char`). Никаких проверок безопасности не производится.

## Даункастинг (Downcasting) с помощью dynamic_cast

> [!info] Сценарий
> Имеется база `Animal` и наследник `Dog`. У собаки есть уникальный метод `bark()`. Указатель `Animal*` не знает про этот метод. Чтобы его вызвать, нужно спуститься вниз по иерархии типов до `Dog*`.

> [!example] Пример безопасного приведения типов
> ```cpp
> #include <iostream>
> 
> // Базовый полиморфный класс
> class Animal {
> public:
>     virtual void makeSomeSound() const { std::cout << "*Звук*\n"; }
>     virtual ~Animal() = default; // Деструктор обязателен
> };
> 
> // Производный класс
> class Dog : public Animal {
> public:
>     void makeSomeSound() const override { std::cout << "Гав!\n"; }
>     
>     // Уникальный метод наследника
>     void bark() const { std::cout << "Я лаю, потому что я собака!\n"; }
> };
> 
> class Cat : public Animal {
> public:
>     void makeSomeSound() const override { std::cout << "Мяу!\n"; }
> };
> 
> int main() {
>     // --- 1. Пример static_cast ---
>     double pi = 3.14159;
>     int intPi = static_cast<int>(pi); // Безопасное отсечение дробной части
> 
> 
>     // --- 2. Пример dynamic_cast (Приведение базы к наследнику) ---
>     Animal* animalPtr1 = new Dog(); // Указатель на базу, но внутри реальная Собака
>     Animal* animalPtr2 = new Cat(); // Указатель на базу, но внутри реальная Кошка
> 
>     // Пытаемся превратить первое животное в собаку
>     Dog* dogPtr1 = dynamic_cast<Dog*>(animalPtr1);
>     if (dogPtr1 != nullptr) {
>         std::cout << "Успех! animalPtr1 — это действительно Собака.\n";
>         dogPtr1->bark(); // Теперь этот метод доступен!
>     } else {
>         std::cout << "Сбой приведения типов.\n";
>     }
> 
>     // Пытаемся превратить кошку в собаку
>     Dog* dogPtr2 = dynamic_cast<Dog*>(animalPtr2);
>     if (dogPtr2 != nullptr) {
>         dogPtr2->bark();
>     } else {
>         // dynamic_cast вернет nullptr, потому что Кошка не является Собакой!
>         std::cout << "Внимание! animalPtr2 — это НЕ Собака. dynamic_cast вернул nullptr. Защита сработала.\n";
>     }
> 
>     delete animalPtr1;
>     delete animalPtr2;
>     return 0;
> }
> ```
