# Спецификатор `= delete` в C++

Начиная со стандарта C++11, в язык был добавлен спецификатор `= delete`. Он позволяет явно запретить вызов определенных функций. При попытке вызвать удаленную функцию компилятор сразу выдаст ошибку сборки.

> [!info] Основное назначение
> Главная цель — жесткий контроль над поведением классов и типов на этапе компиляции.

## Сценарии использования

> [!success] 1. Запрет копирования объектов (Некопируемые классы)
> Если класс управляет уникальным ресурсом (открытый файл, сетевое соединение, потоки), его копирование приведет к багам (когда два объекта пытаются удалить один и тот же ресурс).
> С помощью `= delete` можно явно запретить конструктор копирования и оператор присваивания. Это часто используется при реализации паттерна Singleton или менеджеров ресурсов.
> ```cpp
> NetworkManager(const NetworkManager& other) = delete;
> NetworkManager& operator=(const NetworkManager& other) = delete;
> ```

> [!success] 2. Запрет нежелательных автоматических приведений типов
> C++ склонен неявно приводить типы (например, `double` к `int`). Если функция должна принимать строго `int`, можно удалить ее перегрузку для `double`.
> ```cpp
> void process(int x) { /* логика */ }
> void process(double x) = delete; // Запрет передачи чисел с плавающей точкой
> ```

> [!success] 3. Запрет создания объекта в куче
> Удалив оператор `new`, можно ограничить создание объектов данного класса только стеком (локальные переменные).

## Теория для экзамена (Сравнение с C++98)
До появления `= delete` в стандарте C++98 разработчики прятали копирующие конструкторы в секцию `private` и не реализовывали их.
Современный подход со спецификатором `= delete` значительно лучше, так как:
- Ошибка компиляции получается более явной и понятной.
- Удаленную функцию можно (и нужно) объявлять в секции `public`, благодаря чему компилятор проверит запрет сразу, не спотыкаясь о права доступа.

## Пример в коде

> [!example] Код
> ```cpp
> #include <iostream>
> 
> // --- Пример 1: Некопируемый класс (менеджер ресурсов) ---
> class NetworkManager {
> public:
>     NetworkManager() = default; // Явно просим компилятор создать дефолтный конструктор
> 
>     // ЗАПРЕЩАЕМ конструктор копирования и оператор присваивания
>     NetworkManager(const NetworkManager& other) = delete;
>     NetworkManager& operator=(const NetworkManager& other) = delete;
> 
>     void connect() {
>         std::cout << "Успешное подключение к серверу.\n";
>     }
> };
> 
> // --- Пример 2: Запрет неявного приведения типов ---
> class MathProcessor {
> public:
>     void process(int x) { // Работаем только с целыми числами
>         std::cout << "Обработка целого числа: " << x << "\n";
>     }
> 
>     void process(double x) = delete; // Запрет double
> };
> 
> int main() {
>     NetworkManager manager1;
>     manager1.connect();
> 
>     // NetworkManager manager2 = manager1; // ОШИБКА: попытка вызвать удаленный конструктор!
>     
>     MathProcessor processor;
>     processor.process(10); // Отлично работает
>     // processor.process(3.14); // ОШИБКА: вызов удаленной функции process(double)
> 
>     return 0;
> }
> ```
