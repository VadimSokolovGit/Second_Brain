# Наследование и порядок вызова конструкторов/деструкторов

В C++ при наследовании объект дочернего класса строится "поверх" базового класса. Из-за этого вызовы конструкторов и деструкторов подчиняются строгому зеркальному порядку.

> [!info] Порядок вызова при создании объекта (Конструкторы)
> **Правило:** От базового к производному (Сверху вниз).
> 
> Когда создается объект дочернего класса `Derived`, унаследованный от `Base`:
> 1. Сначала вызывается конструктор базового класса `Base`. Родителю нужно первому инициализировать свои поля, чтобы подготовить фундамент.
> 2. Только после полной отработки конструктора базы запускается конструктор дочернего класса `Derived`, который настраивает свои уникальные поля.

> [!warning] Порядок вызова при удалении объекта (Деструкторы)
> **Правило:** От производного к базовому (Снизу вверх — зеркально).
> 
> Когда объект уничтожается (выходит из зоны видимости или удаляется через `delete`), порядок строго обратный:
> 1. Сначала вызывается деструктор дочернего класса `Derived`. Он зачищает свои уникальные ресурсы (память, файлы).
> 2. Затем автоматически по цепочке вызывается деструктор базового класса `Base`, который убирает за собой общие поля.

## Критическая роль виртуального деструктора
Если происходит удаление объекта наследника через указатель на базовый класс (`Base* ptr = new Derived();`), то без ключевого слова `virtual` перед деструктором базового класса случится беда: компилятор вызовет только деструктор `~Base()`. Деструктор `~Derived()` будет проигнорирован, и все ресурсы дочернего класса останутся висеть в памяти (утечка памяти).

Спецификатор `virtual` заставляет C++ заглянуть в таблицу виртуальных методов и начать удаление именно с «реального» класса-наследника, гарантируя вызов всей цепочки деструкторов.

> [!example] Пример в коде
> ```cpp
> #include <iostream>
> 
> // Базовый класс
> class Base {
> public:
>     Base() {
>         std::cout << "[1] Конструктор BASE\n";
>     }
>     
>     // Виртуальный деструктор — залог правильного порядка при удалении через указатель!
>     virtual ~Base() {
>         std::cout << "[4] Деструктор BASE\n";
>     }
> };
> 
> // Производный класс
> class Derived : public Base {
> public:
>     Derived() : Base() { 
>         std::cout << "[2] Конструктор DERIVED\n";
>     }
>     
>     ~Derived() override {
>         std::cout << "[3] Деструктор DERIVED\n";
>     }
> };
> 
> int main() {
>     setlocale(LC_ALL, "Russian");
> 
>     std::cout << "--- Этап 1: Создание полиморфного объекта ---\n";
>     Base* ptr = new Derived(); 
>     // В консоли мы увидим:
>     // [1] Конструктор BASE
>     // [2] Конструктор DERIVED
> 
>     std::cout << "\n--- Этап 2: Удаление объекта через указатель на базу ---\n";
>     delete ptr;
>     // В консоли мы увидим:
>     // [3] Деструктор DERIVED
>     // [4] Деструктор BASE
> 
>     return 0;
> }
> ```
