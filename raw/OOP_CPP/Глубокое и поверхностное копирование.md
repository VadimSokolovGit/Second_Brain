# Глубокое и поверхностное копирование (Rule of Three)

> [!info] 🎯 Определение
> Когда объект класса содержит внутри себя указатели на динамическую память (выделенную через `new`), дефолтное копирование C++ становится опасным. Возникает концептуальная разница между «поверхностным» (Shallow Copy) и «глубоким» (Deep Copy) копированием.

## 1. Поверхностное копирование (Shallow Copy) — По умолчанию
Если явно не написать свой конструктор копирования, компилятор создаст его автоматически. Он просто побитово скопирует все поля.
- **Что происходит:** Если в классе есть поле `int* ptr`, скопируется сам адрес.
- **Результат:** Два разных объекта смотрят на одну и ту же область памяти в куче.
- **Катастрофа:** Когда первый объект уничтожится, его деструктор вызовет `delete ptr`. Когда второй объект попытается сделать то же самое, программа упадет из-за **двойного освобождения памяти** (double free), либо второй объект будет работать с разрушенными данными (дикий указатель).

## 2. Глубокое копирование (Deep Copy) — Правильный подход
Чтобы объекты были полностью независимы, необходимо реализовать *глубокое* копирование.
При глубоком копировании:
1. Выделяется новый, свой собственный участок памяти в куче для целевого объекта.
2. Сами данные (значения) из старого участка памяти копируются в этот новый участок.

## 💻 Пример в коде

```cpp
#include <iostream>

class MyBuffer {
private:
    int* data;
    int size;

public:
    // Конструктор (выделяет память)
    MyBuffer(int s) : size(s) {
        data = new int[size];
        for (int i = 0; i < size; ++i) data[i] = i * 10;
    }

    // ДЕСТРУКТОР
    ~MyBuffer() {
        delete[] data;
    }

    // КОНСТРУКТОР ГЛУБОКОГО КОПИРОВАНИЯ
    MyBuffer(const MyBuffer& other) : size(other.size) {
        // 1. Выделяем СВОЙ новый кусок памяти
        data = new int[size]; 
        
        // 2. Копируем сами значения
        for (int i = 0; i < size; ++i) {
            data[i] = other.data[i];
        }
    }

    void setElement(int idx, int val) { data[idx] = val; }
    void print() const {
        for (int i = 0; i < size; ++i) std::cout << data[i] << " ";
        std::cout << "\n";
    }
};

int main() {
    setlocale(LC_ALL, "Russian");

    MyBuffer buf1(3); // 0 10 20
    
    MyBuffer buf2 = buf1; // Глубокое копирование
    
    // Если бы копирование было поверхностным, изменился бы и buf1!
    buf2.setElement(0, 999);

    buf1.print(); // 0 10 20 (остался цел)
    buf2.print(); // 999 10 20

    // Деструкторы отработают без ошибок (double free не будет)
    return 0;
}
```

## ⚠️ Правило трех (Rule of Three)
> [!important] Ключевое правило для экзамена
> «Если в классе динамически выделяется память (или управляется другой ресурс), вы **обязаны** явно определить три вещи: 
> 1. Деструктор
> 2. Конструктор копирования 
> 3. Оператор присваивания копированием».
